定义

后缀自动机（suffix automaton，以下简称 SAM）的结构包含两部分，有向无环单词图（directed acyclic word graph，以下简称 DAWG）和前缀树（prefix tree）。SAM 中的每个节点都同时存在于这两个结构中。

我们对一个字符串 $S$ 构造其 $SAM$，称 $S$ 是该 SAM 的母串；下文中提到的「子串」、「前缀」与「后缀」，如非特殊说明，均指母串的「子串」、「前缀」与「后缀」。

记 $|S|$ 为字符串 $S$ 的长度。

应该没什么问题，不过更加科学严谨的定义是：
给定字符串 $s$ 的后缀自动机是一个接受所有字符串 的后缀的最小 DFA （确定性有限自动机或确定性有限状态自动机）。
更精确具体的定义？

  1. 后缀自动机是一张有向无环图。顶点被称作 状态 ，边被称作状态间的 转移 。
  2. 一个状态 为 初始状态 ，它必定为这张图的源点（其它各点均与 连通）。
  3. 每个转移都标有一些字母。从一个顶点出发的所有转移均 不同 。
  4. 一个或多个状态为 终止状态 。如果我们从初始状态 $t_0$出发，最终转移到了一个终止状态，则路径上的所有转移连接起来一定是字符串$S$ 的一个后缀。 $S$的每个后缀均可用一条从 $t_0$到一个终止状态的路径构成。
  5. 后缀自动机是所有满足上述条件的自动机中顶点数最少的一个。

关于DFA是什么在这里就不细说了，好像广义DFA也没什么特殊性质。

DAWG

顾名思义，DAWG 是一个 DAG（有向无环图）。

DAWG 中，除起始节点外，每个节点表示一个或多个子串。

节点之间通过转移边相连，每条转移边上有一个字符。

从起始节点沿着转移边走，每条路径都对应着一个子串，即将走过的边上的字符首尾相连得到的子串（显然多条路径会到达同一个节点上）。

我们称在 SAM 上运行一个字符串 $S$，即为从 DAWG 的起始节点开始，第 $i$ 次沿着字符 $S_i$ 的转移边走，走完 $|S|$次；如果 $S$ 是母串的一个后缀，则称到达的节点为可接受的节点。

Endpos等价类

SAM的核心，重中之重。

定义：考虑字符串$S$的任意非空子串 $T$，我们记 $Endpos(t)$为在字符串 $S$中 $T$的所有结束位置.

有以下重要引理，请确保能够逻辑上严格证明它。

一些重要的引理

引理 1：当且仅当字符串$u$以$w$的一个后缀的形式出现在字符串$s$中时，两个非空子串$u$和 $w$（假设$len_u<len_v$）是$Endpos$等价的

证明：简单的论述和反证都可以。

由定义可知如果$u$以不是$w$后缀的形式出现过的话，就不再是$Endpos$等价的，证明了必要性。

充分性与定义等价。

引理2：设两个子串分别为$u,w$并且$len_u \leq len_w$ ，$u,v$的$Endpos$集合，要么没有交集，要么$Endpos(w)$是$Endpos(u)$的子集。后者满足时，等价于于$u$是$w$的后缀

如果觉得语言描述不好，用数学语言描述：

证明：如果$u$不是$w$的后缀，由定义可知他们没有任何一个共同的右端点，交集为空。
如果$u$是$w$的后缀，在字符串$s$中$w$出现时$u$就会出现，但是$u$出现时$w$不一定出现。
因此$Endpos(w)$是$Endpos(u)$的子集。
其实思想就是：总有个长短，假如有交集并且两个集合都有不属于对方的元素，考虑那个长的字符串里不包含在对方集合的元素，一定包含了短的，因此这个元素实际上包含在另一个的$Endpos$集合里，矛盾！反证完毕

引理3：对于每个$Endpos$等价类，都有一个最长串，其他都是它的后缀，并且长度连续
OI wiki是这么表达的：

证明：显然成立。仔细想一想，我们设最长的长度为$r$ , 最短的长度为$l$，最长的$r$就是最长串长度，十分显然，根据引理1最短串长度$l$满足比它更小的长度的后缀不再仅仅作为最长串的后缀了，再由引理2我们能够得到$[l,r]$长度的后缀均满足条件，因此在$[l,r]$长度的这个等价类的后缀长度在区间连续。

Realise these lemmas , we could continue to learn.

后缀链接Link

这是我当时完全不理解的东西，事实上只是因为我读的太不仔细了而已。。。
所以我决定自己来认真描述一下。

考虑把一个$Endpos$等价类$u$的字串都降序排列，可知每一个都是前一个的前缀。

但并不是所有正整数长度的都在这个等价类里。

因为长度越小的后缀其$|Endpos|$就可能更大，当有一个新位置出现在它某个长度后缀$s$中时，意味着1~$|s|$长度的后缀都不在这个状态里了。

我们连向继续包含这些后缀并且最长长度最大的状态$v$，由于每个字串都出现了所以有:
$min(u) = max(v) + 1$
这样通过后缀链接我们成功将一个前缀所有后缀链接到了一起。

每个状态有唯一的前驱，因此所有状态的后缀链接构成了树形结构。也许我们可以称他为前缀树，它的根节点是$t_0$，$Endpos(v)$是$Endpos(link(v))$的子集。
这是OIWIKI的引理4，5，我们又解决了一个重要概念。

现在我们做一个小结，就可以进入最重要的线性构造并应用了。

  1. $S$的字串可以被分成若干个$Endpos$等价类。
  2. 每个节点是包含某一前缀若干长度单调递减的后缀，并由后缀链接成长度从1到前缀长度的后缀的并集。

首先，如果不存在这样的 $v_3$，满足 $v_3$ 有字符 $c$ 的出边，则需要将 $u$ 的后缀链接连向起始节点$ \text{start}$，因为新出现的 $|S| + 1$ 个后缀都需要节点 $u$ 来表示，即 $|\min(u)| = 1$，而为了满足前缀树是一棵树，需要将 $u$ 的父节点置为树根。

此时我们不能将 $u$的后缀链接连向 $d$，因为 $d$中的字符串不全是 $u$的后缀

我们需要把等价类拆开。对于长度大的那部分后缀链接连向长度小的。
举个例子， $v_3$中有子串$lyxtst$ ， $v_2$ 中有子串$aalyxtsts$， $t$中有子串$lllyxtst$ ， $u$中有子串$aalyxtsts$， $d$中有子串$lyxtsts$和$llyxtsts$，因为 $llyxtsts$不是 $aalyxtsts$ 的后缀，所以$ u$的后缀链接不能连向 $d$。否则就会多出本不存在的子串。

我们需要将$ d$拆成两个点，一个点$n$表示长度小于等于 $|\max(v_3)| + 1$的子串，另一个点 $o$表示长度更大的子串.

原有的 $d$中，仅有长度小于等于$ |\max(v_3)| + 1 $的子串是$ v_3 $与 $v_4$ 中的字符串加上一个字符转移而来的，所以$ v_3 $与$ v_4 $ 的字符$ c$ 的转移边应该连向 $n$，而其它子串均为其它节点转移而来的，所以其它节点的转移边应该连向 $o$，并且$\text{next}(n) $ 应为原有的 $\text{next}(d)$，并且 $\text{next}(o) = n$，然后新节点$ u $ 的后缀链接$ \text{next}(u) = n $

复杂度证明

接下来我们证明这个算法的时空复杂度，它可以达到线性的空间复杂度，并在字符集为常数的情况下，达到线性的时间复杂度。

空间复杂度

整个结构包含两部分 —— DAWG 和前缀树，我们需要分别证明节点数和 DAWG 的边数是线性的，并且与字符集无关。

首先，每次加入节点时，最多新建两个节点，所以显然节点数的上界是 $2|S|$，这一部分的复杂度得证。（更严格的上界减个常数没啥意义）

对于 DAWG 的转移边，我们可以再将其分为两部分 —— 我们以起始节点为根，对 DAWG 建立一棵生成树，树边的数量一定是线性的，接下来只需要考虑非树边的数量。

我们定义集合 $E$ 表示所有非树边，集合 $S$ 表示串的所有后缀。如果我们能找到一个映射$ f: E \rightarrow S$，使得对于任意的 $a, b \in E$，若$ f(a) = f(b) $，一定有 $a = b$(即$f$为单射)，即可证明 $|E| \leq |S|$。

对于两个节点 $u, v$ ,显然只可能存在一条非树边 $u \rightarrow v$或$ v \rightarrow u $，假设存在 $u \rightarrow v $，那么我们找到下面三段字符串：

  1. 从生成树的根沿着树边走到$u$，将经过的所有转移边上的字符按顺序首尾相接（因为是沿着树边，所以这个串是唯一的）；
    2 . $u \rightarrow v$ 这条转移边上的字符；
    3 . 从 $v$开始，沿着字符字典序最小的转移边（可以是树边或非树边）走，直到走到一个可接受的节点$w$，将经过的所有转移边上的字符按顺序首尾相接。

设这三段拼接得到得到的字符串为 $s$ .因为这三段都是唯一的，所以 $s$ 是唯一的，记$f(u \rightarrow v) = s$；又因为整条路径是从起始节点走到一个可接受的节点，所以 $s$ 一定是母串的一个后缀，并且，这条非树边 $u \rightarrow v$ 是运行字符串 $s$时经过的第一条非树边。

  1. 每条非树边 $u \rightarrow v$ 都能对应到一个 $s$
  2. 每个被对应到的 $s$ 都能对应到唯一一个 $u \rightarrow v$。

这样我们建立了非树边与后缀之间的一一映射，非树边数量也是线性的。

我们就神奇的证明了SAM的空间复杂度是线性的。

时间复杂度

只需要证明两部分：

  1. 在后缀链接上将没有转移边$c$的状态连一条边直到出现第一个状态有转移边$c$（我们会说明跳到根没有这样的状态是平凡情况）
  2. 如果是第三种构造情况，将那些转移边重定向。

对于第一种，由于每次操作对应一条新转移边，又因为我们证明转移边数是线性的，得证。

当然也可以向下面这样证明。

我们关注$max(link(las))$

由于一定会进行一次，所以我们设$v$表示$link(las)$

  • 循环一定会执行第一次（$v$ 一定会被赋值为 $link(last)$），也就是说，循环终止时 $max(v)$最大为 $max(link(last))$，并且，$max(v)$ 经过常数时间即可从更大的值变为 $max(link(v))$；
  • 之后循环每进行一次迭代，都会使 $max(v)$至少减少 1（因为 $max(link(v))$ 一定小于 $max(v)$）；
  • 循环结束后，$link(u)$ 被赋值为一个 $max$ 为 $max(v) + 1$ 的节点，而 $last$ 被赋值为 $u$，也就是说，$max(link(last))$ 的值变为 $max(v)+1$。

这非常重要，我们可以聚合分析。

每次暴力修改的边数是不确定的，但是由于$len(link(link))$长度每次插入后只能加1，长度递减但是非负，所以这一部分时间复杂度为$O(n)$

还不明白的话再画一个新节点，然后你就会发现转移后缀链接的时候是可以势能分析的。它最多递减$n$。

所以之所以一开始我觉得不可证是因为没想到$N$和$R$的关系。。。仔细想想确实是这样的。

也就是说$len(link(link))$单调递减吗，其实表达的就是$R$长度的分析，毕竟我们只需要常数的复杂度即可到达$link->link$

这样我们就证明了SAM的整个构造算法是$O(n)$的。

Lexicographically k-th substring

We calculate the different substrings of each status $u$ as $f_u$ in $O(n)$ time .

then enumerate each char $x$ .

if $k$ $<$ $\sum{i=1}^{x}f{u,i}$ , $x = x + 1$

else $k -= \sum{i=1}^{x}f{u,i} , x = tr(u,i)$

Yeah it is a simple divide and conquer problem.

For each query the complexity of time is $O(n\sum)$ , which $\sum$ denotes the size of char

Repeat $|S|$ times we got it.

Complexity : $O(n)$

Shortest non-appearing string

seems to be hard!

let’s $f_u$ means the least number of char need to be added for Status $u$(The ans of status $u$)
if there is a transit char belongs to alphabet , but the status $u$ did not exists , then $f_u = 1$

else $fu=1 + \min{edge(u,v,c)}{f_v}$

The answer is $f_{t_0}$
